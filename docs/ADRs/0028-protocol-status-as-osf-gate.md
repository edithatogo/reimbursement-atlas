# ADR 0028: Treat protocol completeness as an OSF release gate

## Status

Accepted.

## Context

OSF is intended to hold detailed protocols, reports, appendices and future preprint materials. A list of planned protocols is insufficient for release governance.

## Decision

Generate protocol-status records from the research-question registry and protocol/report files. OSF upload and publication candidates should reference this table before treating analyses as preregistered or report-ready.

## Consequences

Research questions become auditable work units with missing-section and completeness signals. This also creates GitHub issue/project fodder for protocol completion and review.
